# Design System Specification: Industrial Realism & Tactile Precision

## 1. Overview & Creative North Star: "The Analog Machine"
The Creative North Star for this design system is **The Analog Machine**. We are moving away from the "flatness" of the modern web to embrace the weight, friction, and reliability of mid-century industrial instrumentation. This is not just "neumorphism"—it is **Industrial Realism**.

The goal is to make the user feel as though they are operating a physical chassis. We achieve this through:
*   **Tactile Precision:** Every element must feel like it was machined from a solid block of material.
*   **Mechanical Reliability:** Interactions must mimic physical spring physics—actions have weight, resistance, and consequences.
*   **Intentional Asymmetry:** Break the standard digital grid by placing "screws" in corners and using "ventilation slots" to create organic breaks in the paneling.

---

## 2. Colors & Materiality
The palette is rooted in the "Chassis" and "Panel" tones, designed to mimic powder-coated steel and bead-blasted aluminum.

### Palette Strategy
*   **Chassis (`#e0e5ec`):** The master structural frame. Everything sits within this.
*   **Panel (`#f0f2f5`):** The primary work surface. Use this for the main interaction areas.
*   **Recessed (`#d1d9e6`):** Used for "etched" areas, gutters, and input wells.
*   **Safety Orange (`#ff4757` / `primary`):** Reserved exclusively for critical actions, warnings, and active status indicators.
*   **Primary Text (`#2d3436`):** High-contrast, legible "ink" on the metal surface.

### The "No-Line" Rule
**Prohibit 1px solid borders.** In this system, boundaries are created by lighting and depth. To separate sections, transition from a `surface` to a `surface-container-high` or use a "Recessed" shadow. A line is a failure of geometry; a shadow is a triumph of form.

### Surface Hierarchy & Nesting
Treat the UI as a physical assembly:
1.  **Level 0 (Chassis):** The outermost container.
2.  **Level 1 (Panel):** Bolted onto the chassis (use `surface-container-low`).
3.  **Level 2 (Interface Elements):** Raised buttons or recessed inputs.
Use the `surface-container` tiers to create this "machined" look. An inner container should always be `surface-container-highest` to appear closer to the user, or `surface-dim` to appear sunken.

## 3. Typography: The Technical Manual
Our typography balances human-centric UI with cold, technical precision.

*   **UI & Body (Inter):** Used for all instructional text and labels. It provides the "human" interface to the machine.
    *   *Body-md:* `0.875rem` / Inter. Use for standard descriptions.
    *   *Title-sm:* `1rem` / Inter Bold. Use for panel labels.
*   **Technical Metrics (JetBrains Mono):** This is our "Readout" font. Any value that changes (numbers, timestamps, data) must use this to imply it is being generated by the machine's internal logic.
*   **Display (Space Grotesk):** Used for high-level headers. Its geometric rigidity reinforces the industrial aesthetic.

---

## 4. Elevation & Depth: The Physics of Light
All depth is calculated based on a **45-degree top-left light source**.

### The Layering Principle (Neumorphic Shadows)
Instead of standard drop shadows, use dual-shadow stacks:
*   **Base Lift:** Light shadow (`-6px -6px 12px #ffffff`) and Dark shadow (`6px 6px 12px #d1d9e6`).
*   **Pressed/Recessed:** Light shadow (`inset -2px -2px 5px #ffffff`) and Dark shadow (`inset 2px 2px 5px #d1d9e6`).

### Ambient Lighting & Textures
*   **Corner Screws:** Every major panel corner should feature a `0.5rem` radial gradient (from `#d1d9e6` to `#f0f2f5`) to mimic a recessed hex screw.
*   **Ventilation Slots:** Use 3px high rectangles with a `Recessed` inner shadow to break up large empty areas of a panel.
*   **CRT Scanlines:** Use a subtle linear-gradient overlay (`rgba(18, 16, 16, 0.05)`) on all image placeholders or data visualizations to simulate an old-school monitor.

## 5. Components

### Buttons (Mechanical Actuators)
*   **Primary (Safety Orange):** High elevation. On hover, the elevation decreases slightly. On click, use `Pressed` shadows and a `1px` downward Y-axis translation to simulate mechanical spring compression.
*   **Secondary (Chassis Tone):** Subtle lift. Use for secondary controls.
*   **Toggle Switches:** Use a physical "rocker" look. One side `Recessed`, one side `Base Lift`.

### LED Indicators
*   Replace standard "status dots" with glowing LEDs.
*   **Active:** `Safety Orange` with a `0 0 8px #ff4757` outer glow.
*   **Inactive:** `surface-dim` with an inset shadow.

### Input Fields (Machined Wells)
*   Inputs must always be **Recessed**. The "glass" of the input should feel like it is 2mm below the panel surface.
*   Use `JetBrains Mono` for the typed text.

### Cards & Lists
*   **Forbid Divider Lines.** Separate list items using a `0.2rem` gap and subtle background shifts (Panel to Chassis).
*   **Nesting:** A card sitting on a panel should have a `Base Lift`. A card inside a sidebar should be flush but defined by a `Ghost Border` (outline-variant at 15% opacity).

---

## 6. Do’s and Don’ts

### Do:
*   **Do** use "Mechanical Spring Physics" (Damping: 20, Stiffness: 300) for all transitions.
*   **Do** place "Screws" (radial gradients) in the four corners of the main application container.
*   **Do** use `JetBrains Mono` for any number that looks like data.
*   **Do** respect the 45-degree light source globally; if one shadow is off, the "realism" breaks.

### Don’t:
*   **Don’t** use pure black `#000000` for shadows. Always use a tinted version of the background (`#d1d9e6`).
*   **Don’t** use rounded corners larger than `0.75rem (xl)`. Industrial machines are blocky and rigid.
*   **Don’t** use animations that "fade." Physical objects don't fade; they slide, click, or pop.
*   **Don’t** use high-contrast 1px borders. If you need a border, use the "Ghost Border" at 10% opacity.
